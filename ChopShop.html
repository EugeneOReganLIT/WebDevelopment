The Chop Shop Tool Kit

Like any skilled professional (or dedicated hobbyist), there's always the need to have small examples of previous work at hand to assist in future projects. This could be in the form of notes, sketches, labelled prototypes or in the case of the programmer, a store of carefully commented snippets of code. This code will be short but very focused and commented (which may be removed in the commercial release of the code).

Accordingly, I will be providing sample code that will be developed in an incremental manner. This will be done to demonstrate the importance of disciplined programming and keeping things simple. Time permitting, I will also provide videos of the more important examples and will work on the more popular requested code.

The Panucci Chop Shop Challenge

Like playing the guitar, unless you're practising coding frequently, your skills won't evolve. So, with that in mind, whether you're stuck on basic strings or fancy yourself as the Rory Gallagher of recusion, there should be something below to get you rocking.

Challenge No.1 - November 2017

The challenges below will involve the following components, so get reading those books, scanning websites and terrorising the forums. I'll throw out a few hints from time to time, if people are stuck. The main aim is that you work out the algorithm rather than the code. But most of all, have a bit of fun whilst doing them.

  • Arithmetic
  • Loops
  • Parsing
  • Arrays

Assume all input is numeric and positive for now. Obviously if you can develop them, do!

Absolute Beginners - Opening Chords

Write a program that will take in two numbers from the screen as input, add them and output the result.

Improve the program by allowing the user to continue this process (entering 2 numbers and seeing the result on screen):

  • A set number of times, defined at the start of the program (either hard coded or entered by the user).
  • Until the user inputs a specific number or character e.g. a negative number.

Stuck in the Middle with you – Barre Chord Bridge

Write a program that will take in a number that has more than one digit i.e. a number greater than 10. The program will then add up the sum of the individual digits of that number. For example, if the user enters 37, the program will perform 3 + 7, giving a total of 10. Keep the numbers whole and positive for now.

Master of Puppets – Knuckle Bustin’ Closer

Write a program that will take in a number from screen. Without the use of either divisor operands, determine if the number is divisible by 9. It is possible and relies on a mathematical property of the number 9.
