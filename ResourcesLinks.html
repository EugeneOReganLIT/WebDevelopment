Useful Equipment

The following is a list of non-computer related equipment that has proven to be helpful in the initial composition of code. The list is confirmation that time away from the computer and compiler is necessary in order to properly ascertain all details of the presented scenario.

  • A4 sketchpads - Good for rough work and diagrams.
  • Whiteboards - While maybe seen as overkill, a whiteboard can be very helpful for group discussions and working out problems initially.
  • A4 foolscap paper - To write down and refine the rough code.
  • Multi-coloured pens - To denote different elements in the code e.g. comments, method calls, etc.
  • Snipping Tool / PrintScreen - Good for compiling notes in a classroom environment or when being shown how to do a task in a compiler.
  • Mobile Phone- To take pictures of code on a whiteboard or to double as a dictaphone.

Books

It is always beneficial to have a book at hand when programming. As well as being easier to read than webpages, you can mark sections for quick reference.

Getting the latest versions of the books ... is it needed?

It is important to note that as there are 9 revisions of Java, there maybe a strong compulsion to get the most recently published books in order to be up to date. This is totally unecessary. Books covering previous versions of Java will be sufficient for any beginner / intermediate programmer. So with that in mind, here is a list of good publications:

  • Beginning Programming with Java For Dummies, 3rd Edition,Burd, B.A. - Another in the popular series of books aimed particularly at beginners.
  • OCA Java SE 8 Programmer I Certification Guide 1st Edition, Gupta, M. - Although written for the OCA exams, Gupta covers the subjects expertly and in an accessible manner. An excellent author.
  • Java in 21 Days, Sams Teach Yourself (Covering Java 9) (8th Edition) Cadenhead, R. - A good reference text, in it's 8th guise. It is well organised, with distinct sections and clear path of progression.
